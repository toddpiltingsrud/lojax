custom attributes

lojax uses custom attributes to allow any clickable element to trigger calls to a web server.

@data-method

An element must at least have a "data-method" attribute to be handled by lojax. This attribute specifies what type of call to make:

<a href="#howitworks.html" data-method="ajax-get">How it Works</a>

The following methods are supported:

  • get
  • post
  • ajax-get
  • ajax-post
  • ajax-put
  • ajax-delete
  • jsonp

@data-action

You can use the data-action attribute instead of href:

<button data-method="ajax-get" data-action="#howitworks.html">How it Works</button>

@data-form

The data-form attribute is a jQuery selector that specifies a collection of input elements:

<input type="text" name="FirstName" value="Todd">
 
<input type="text" name="LastName" value="Piltingsrud">
 
<button data-method="ajax-post" data-action="Create"  data-form="[name]">Submit</button> 

If you use an ajax method with the data-form attribute, lojax passes the value of data-form to jQuery's serialize function. If you use a non-AJAX method (POST, GET, etc), lojax creates a hidden form for the matching elements and calls the submit function on it. This frees you to put input elements anywhere on the page and submit any queryable combination of them to the server.

@data-model

This attribute expects either a JavaScript object or a JSON string. Similar to the data-form attribute, submitting via AJAX serializes the model before making the call. Submitting via non-AJAX methods creates a hidden form and populates it with the model's object names and values and then submits the form.
